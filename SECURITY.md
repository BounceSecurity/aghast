# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |

Only the latest released version of aghast receives security updates.

## Reporting a Vulnerability

If you discover a security vulnerability in aghast, please report it responsibly by emailing **info@bouncesecurity.com**.

Please include:

- A description of the vulnerability
- Steps to reproduce the issue
- Any relevant logs, screenshots, or proof of concept

We will acknowledge your report within 5 business days and aim to provide a fix or mitigation plan within 30 days, depending on severity.

Please do **not** open a public GitHub issue for security vulnerabilities.

## Scope

This policy covers the aghast CLI tool and its published npm package (`@bouncesecurity/aghast`). Security check definitions maintained in separate configuration repositories are out of scope for this policy.
